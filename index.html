The finding

Alerts propagate geographically. Peripheral western oblasts get advance signal as threats cross the country and are meaningfully predictable; strategic direct targets like Kyiv are struck with little lead time and look near-random from a single-region view.
Predictable

Lviv, Zhytomyr (ROC-AUC ~0.66). Small western oblasts gain most from neighbour features (Ivano-Frankivsk +0.039 AUC) — exactly where propagation theory predicts.

Near-random

Kyiv City (ROC-AUC ~0.55). Direct targets gain nothing from neighbours (+0.001). Frontline regions (Kharkiv, Dnipro) show high F1 only because alerts are active 40–65% of the time.

Reproduces a published result

Matches arXiv:2411.14625 — a region's alert status depends heavily on its adjacent regions — and localizes where that dependence is strong vs weak.

How it was built

  • Task: binary classification — will an alert begin in the next 3 hours? Chosen over exact-time forecasting (false precision, hard to validate).
  • Data: Vadimkin volunteer dataset, oblast-level, ~101k records, UTC. Permanent-siren regions (Luhansk, Crimea) dropped; naive end-times kept for events but excluded from duration features.
  • Validation: strict chronological train / val / test split — never random, which would leak the future into the past.
  • Metrics: precision, recall, F1, ROC-AUC, confusion matrix — recall-weighted, because a missed alert costs more than a false alarm.

Leakage audit — the core guarantee

Three automated checks prove no feature sees the future. All pass.
  • PASS — Reconstruction independence: deleting all future data changed past features by 0.00e+00 across 18,824 hours.
  • PASS — Target is genuinely future: 0 mismatches vs independent timestamp recomputation.
  • PASS — No feature trivially encodes the target: max |correlation| = 0.21.
